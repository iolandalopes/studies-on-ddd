## Domain-Driven Design (DDD)

**Domain-Driven Design (DDD)** é uma abordagem de desenvolvimento de software focada no **domínio do negócio** e na **colaboração entre especialistas de negócio e desenvolvedores**. Seu objetivo é criar sistemas mais alinhados às regras, processos e linguagem do negócio.

### Conceitos principais

- **Domínio**: área de conhecimento ou atividade que o software atende.
- **Linguagem Ubíqua**: linguagem comum entre time técnico e negócio, usada no código e nas discussões.
- **Bounded Context**: limites bem definidos onde um modelo de domínio é válido.
- **Entidades**: objetos com identidade própria ao longo do tempo. Entidades possuem regras de negócio. Devem atender ao princípio da autovalidação.
- **Objetos de Valor (Value Objects)**: objetos imutáveis definidos apenas por seus atributos.
- **Agregados**: conjunto de entidades e objetos de valor tratados como uma unidade.
- **Repositórios**: abstração para acesso e persistência de dados.
- **Serviços de Domínio**: regras de negócio que não pertencem naturalmente a uma entidade.


Este repositório tem como objetivo estudar e praticar os conceitos do DDD por meio de exemplos e experimentos.
